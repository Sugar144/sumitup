# Introduction to Systems Administration — Knowledge Extraction

**Source:** *Introduction to systems administration* — Miquel Colobran Huguet, Javier Panadero Martínez (FUOC/UOC, 2026)

---

## 1. IT Systems and the Organisation

| # | Concept | Type | Description |
|---|---------|------|-------------|
| 1 | IT Department Perspectives | Framework | Four views: **Organisational** (department with resources), **Technical** (servers, networks, workstations), **User** (tool for improving work), **Management** (query database for decision-making). |
| 2 | Effectiveness vs Efficiency | Definition | Effective = produces expected results. Efficient = achieves the task within reasonable time and resources. |
| 3 | IT Manager | Role | Head of the IT department; final decision-maker regarding the organisation's IT system. Also called IT Director, Head of IT. |

## 2. IT Systems: Elements

| # | Concept | Type | Description |
|---|---------|------|-------------|
| 4 | IT Department Composition | Structure | Three core elements: **IT Manager**, **Technical Support Staff**, **Physical IT System**. |
| 5 | Hardware: Servers | Component | Remote computers in the network storing programs and data. Multiple servers specialise in different functions to increase control and reduce risk. |
| 6 | Hardware: Workstations | Component | The computer assigned to each user (worker). High volume in most organisations. PCs, Macs, Linux, laptops, terminals. |
| 7 | Hardware: Network Cabling & Electronics | Component | Physical cables + network control electronics (hubs, routers, firewalls, switches). |
| 8 | Software: Basic | Component | Operating system and basic communications applications on servers. |
| 9 | Software: Office | Component | Office suites — spreadsheet, word processor, database, agenda. |
| 10 | Software: Communications | Component | E-mail, internet browser (Edge, Firefox, Chrome), video calls (Teams, Hangouts). |
| 11 | Software: Specific Applications | Component | Organisation-/department-dependent: invoicing, accounting, graphic design, production control, payroll. |

## 3. IT System Staff — Roles

| # | Concept | Type | Description |
|---|---------|------|-------------|
| 12 | Server Administration | Role | Installing, maintaining and servicing servers and IT infrastructure. Must ensure quick recovery on failure and keep servers running 24/7. |
| 13 | User Administration | Role | Taking care of users' needs and maintaining their desktops. Most visible part of the system. |
| 14 | Network Administration | Role | Ensuring the physical network works properly, monitoring it, upgrading it, and extending it to all areas. |
| 15 | Data Management | Role | Maintaining the integrity of the organisation's information — on servers and scattered workstations. Includes large databases and linking stored information. |
| 16 | Web Administration | Role | Maintenance of the web server and often the content (webpages). Maintains overall server consistency. |
| 17 | Security Management | Role | Securing information, physically protecting equipment, preventing internet-borne attacks. Security is present in ALL roles. |
| 18 | Development | Role | Developing or acquiring specific software for the organisation. Often outsourced to consultancies. Alternatives: purchasing standard software (e.g. SAP) or outsourcing. |
| 19 | IT Manager (detailed) | Role | Most senior IT role. Liaises between organisational needs and department work. Decides software purchases, server installations, medium/long-term IT strategy. |
| 20 | Essential Roles | Principle | Minimum required: **IT Manager**, **User Administrator**, **Server Administrator** — can be performed by a single person. |
| 21 | Role Overlapping | Principle | Roles have defined tasks but work in coordination. Tasks often cross over between roles (e.g., registering users may involve both user admin and server admin). |
| 22 | Organisation Chart | Structure | Depends on organisational culture, background, habits, and technology. Technology can introduce changes due to shifting needs. |
